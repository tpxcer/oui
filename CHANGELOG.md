# 更新日志

所有正式版本都应写中文更新说明。发布前请同步更新：

- `CHANGELOG.md`：保留历史版本说明。
- `config/release-notes/latest.md`：写入当前待发布版本说明，GitHub Release 会自动使用它作为发布正文。

## 2026.7.21

- 更新过程中只让箭头向下移动并淡出循环，圆形背景和托盘不旋转，更直观地表达正在下载。

## 2026.7.20

- 优化限制为 1 个 IP 时的网络切换：出现第 2 个 IP 后，系统会统计原占位 IP 在当前入站端口连续 20 秒的双向流量。
- 原占位 IP 在采样期内流量低于 100 KiB 时，自动释放原 IP 并由新 IP 接管，不再因手机流量与 Wi-Fi 切换产生残留封禁。
- 原占位 IP 流量达到或超过 100 KiB 时，保持原 IP 并按现有规则掐断新 IP；三个及以上 IP 同时出现时继续严格执行原限制。
- 正常 IP 接管不会发送“超出 IP 上限，已掐断”通知；流量统计同时覆盖 TCP、UDP、上传和下载。
- 增加采样阈值、接管、并发超限及异常清理回归测试。

## 2026.7.19-1

- 客户端启用开关改为专用接口，只修改启用状态，不再覆盖凭据、流控、IP 限制、流量或期限等配置。
- 同一客户端关联多个入站时，启用状态会同步应用到全部关联入站。
- 增加前后端回归测试，确保反复关闭和启用后原连接信息保持不变。
