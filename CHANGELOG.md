# 更新日志

所有正式版本都应写中文更新说明。发布前请同步更新：

- `CHANGELOG.md`：保留历史版本说明。
- `config/release-notes/latest.md`：写入当前待发布版本说明，GitHub Release 会自动使用它作为发布正文。

## 2026.6.15

- 修复命令行证书设置不一致：`x-ui setting -webCert/-webCertKey` 现在会真正写入面板证书路径，仍兼容 `x-ui cert -webCert/-webCertKey`。
- 修复 Web 访问路径带尾斜杠时可能访问异常的问题：内部统一按不带尾斜杠保存，路由兼容 `/abc`、`/abc/` 和 `/abc/panel/`。
- 优化更新脚本的 SSL 提示：证书申请失败或跳过 SSL 时不再误显示 HTTPS 成功信息。
- 优化更新脚本 SSL 配置选项，更新时也可以明确选择跳过 SSL 并绑定到本机地址。
- 优化防火墙管理：按 UFW 规则序号批量删除时改为倒序删除，避免序号变化导致误删。
- 优化 IP Limit 卸载流程：完整卸载 Fail2Ban 前增加明确二次确认。

## 2026.6.11-1

- 调整 Telegram 后台更新通知：点击一键更新后的“已开始后台更新”和“更新成功”反馈只显示更新状态，不再重复展示本版本更新内容。
- 保留 Telegram 更新菜单中的本版本更新内容展示，方便更新前查看更新说明。
- 同步更新项目中文说明，明确不同更新入口的展示规则。

## 2026.6.11

- 新增 Hysteria2 端口跳跃：可在入站配置里填写 UDP 跳跃端口范围，OUI 自动部署 nftables/iptables 转发规则。
- Hysteria2 分享链接和订阅链接自动携带 `mport`，导入带 `mport` 的链接时会回填端口跳跃配置。
- Telegram 和面板一键创建 Hysteria2 时会自动生成跳跃端口范围，并尝试放行真实端口和跳跃范围。
- 保存、启停、删除入站以及面板启动时会自动同步端口跳跃规则；启用入站时会检查跳跃范围冲突。
- nftables 端口跳跃规则新增计数器，便于在 VPS 后台确认实际命中情况。
- 优化入站表单部分中文文案，并把 Hysteria2 端口跳跃入口放到基础配置的端口区域，创建节点时更容易找到。

## 2026.6.10-1

- 调整 Telegram 客户端下线通知规则：客户端连续 10 分钟没有新增流量后才触发下线通知。
- 如果 10 分钟窗口内重新产生流量，会重新计算下线等待时间，避免短暂断流误报下线。
- 下线通知里的“下线时间”改为连续无流量开始时间，不再使用通知实际发送时间。
- 下线通知里的在线时长同步按真实下线时间计算。

## 2026.6.10

- 更新项目中文介绍，移除已经失效或过期的功能描述。
- 新增中文发布说明来源，后续每个版本都应先更新本文件和 `config/release-notes/latest.md`。
- 面板检测更新时会展示 GitHub Release 中的本版本更新内容。
- 面板一键更新确认弹窗会同步展示本版本更新内容。
- Telegram 更新菜单、开始更新提示和更新成功反馈会同步展示本版本更新内容。
