# 更新日志

所有正式版本都应写中文更新说明。发布前请同步更新：

- `CHANGELOG.md`：保留历史版本说明。
- `config/release-notes/latest.md`：写入当前待发布版本说明，GitHub Release 会自动使用它作为发布正文。

## 2026.6.10-1

- 调整 Telegram 客户端下线通知规则：客户端连续 10 分钟没有新增流量后才触发下线通知。
- 如果 10 分钟窗口内重新产生流量，会重新计算下线等待时间，避免短暂断流误报下线。
- 下线通知里的“下线时间”改为连续无流量开始时间，不再使用通知实际发送时间。
- 下线通知里的在线时长同步按真实下线时间计算。

## 2026.6.10

- 更新项目中文介绍，移除已经失效或过期的功能描述。
- 新增中文发布说明来源，后续每个版本都应先更新本文件和 `config/release-notes/latest.md`。
- 面板检测更新时会展示 GitHub Release 中的本版本更新内容。
- 面板一键更新确认弹窗会同步展示本版本更新内容。
- Telegram 更新菜单、开始更新提示和更新成功反馈会同步展示本版本更新内容。
